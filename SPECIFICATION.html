1. 프로젝트 개요

1.1 목적

BTS의 영문 가사를 활용하여 영어를 학습할 수 있는 모바일 애플리케이션 개발

1.2 주요 기능

  • Spotify 연동: BTS 전체 곡 목록 조회 및 30초 미리듣기
  • 실시간 가사 싱크: 재생 시간에 맞춰 가사 표시
  • 영어 단어 학습: 가사 속 단어 클릭 시 사전 검색
  • 크로스 플랫폼 지원: iOS, Android, Web 동시 지원
  • OAuth 인증: Spotify 계정 연동

1.3 대상 사용자

  • BTS 팬
  • 영어 학습자
  • 음악으로 언어를 배우고 싶은 사용자

1.4 프로젝트 범위

항목 범위
음악 소스 Spotify (30초 미리듣기)
가사 소스 lrclib.net API
사전 API Free Dictionary API
인증 Spotify OAuth 2.0 PKCE
배포 플랫폼 iOS, Android (Expo), Web

4. API 연동 사양

4.1 Spotify Web API

📌 목적

BTS 곡 목록 조회 및 메타데이터 획득

주요 엔드포인트

메서드 엔드포인트 설명
GET /artists/{id}/albums 아티스트 앨범 조회
GET /albums/{id}/tracks 앨범 트랙 조회
GET /search 곡 검색

인증

Authorization: Bearer {access_token}

응답 데이터 예시

{ "id": "3mAF4MXWFCxCDBRZcEPr98", "name": "Spring Day", "artists": [{"name": "BTS"}], "preview_url": "https://...", "duration_ms": 245573, "album": {"images": [{"url": "..."}]} }

4.2 lrclib.net API (가사)

📌 목적

타임스탬프 포함 동기화된 가사 조회

엔드포인트

GET https://lrclib.net/api/search?track_name=Spring%20Day&artist_name=BTS

응답 필드

필드 타입 설명
syncedLyrics string [mm:ss.xx]text 형식의 동기화 가사
plainLyrics string 타임스탬프 없는 일반 가사

4.3 Free Dictionary API (단어 사전)

📌 목적

영어 단어 정의 및 발음 조회

엔드포인트

GET https://api.dictionaryapi.dev/api/v2/entries/en/{word}

응답 샘플

{ "word": "spring", "phonetic": "/sprɪŋ/", "meanings": [{ "partOfSpeech": "verb", "definitions": [{ "definition": "move rapidly upward...", "example": "he sprang up..." }] }] }

5. 인증 및 보안

5.1 Spotify OAuth 2.0 PKCE 플로우

1. 사용자 로그인 요청 ↓ 2. Code Verifier 생성 ↓ 3. Code Challenge 생성 (SHA-256) ↓ 4. Authorization URL로 리다이렉트 ↓ 5. 사용자 승인 ↓ 6. Authorization Code 수신 ↓ 7. Code Verifier와 함께 Token 요청 ↓ 8. Access Token 획득 ↓ 9. AsyncStorage에 토큰 저장

5.2 토큰 관리

저장 위치

  • AsyncStorage: spotify_token (크로스 플랫폼)
  • 저장 정보: access_token + expiry_time

토큰 갱신 정책

  • 만료 시간 체크: Date.now() > expiry_time
  • 만료된 토큰: 자동 삭제 및 재인증 필요

5.3 보안 고려사항

항목 적용 현황
PKCE 사용 ✅ 적용됨
HTTPS ✅ Spotify API 강제
Client Secret 없음 ✅ PKCE로 대체
환경 변수 분리 ✅ .env 파일 사용
⚠️ 주의: .env 파일이 .gitignore에 포함되어 있으므로 민감한 정보가 저장소에 노출되지 않습니다.

6. 데이터 흐름도

6.1 앱 초기 시작 흐름

App.tsx (마운트) ↓ initializeApp() 호출 ↓ getStoredToken() - AsyncStorage에서 토큰 확인 ↓ 토큰 있음? ├─ YES → setToken() → HomeScreen 이동 └─ NO → LoginScreen 표시

6.2 곡 선택 후 재생 흐름

HomeScreen: 곡 선택 ↓ handleSelectTrack(track) ↓ 1. 현재 트랙 설정 2. PlayerScreen으로 이동 3. getLyrics(trackName, artistName) 호출 ├─ lrclib.net API 요청 ├─ 파싱 (타임스탬프 추출) └─ setLyrics() 4. Audio.Sound.createAsync() - 오디오 초기화 5. sound.playAsync() - 재생 시작 6. setOnPlaybackStatusUpdate() - 진행률 추적

6.3 가사 동기화 흐름

Audio 재생 중 ↓ 250ms 마다 setCurrentMs() 호출 ↓ currentMs와 가사 라인의 startMs/endMs 비교 ↓ 현재 라인 인덱스 계산 ↓ LyricsView에서 하이라이트 및 스크롤 ↓ 가사 끝나면 자동으로 다음 곡 재생

6.4 단어 검색 흐름

사용자: 가사의 단어 클릭 ↓ handleWordPress(word) ↓ getWordDefinition(word) ├─ 정규식으로 단어 정제 └─ Free Dictionary API 호출 ↓ 응답 파싱 ↓ VocabCard로 표시 (뜻, 발음, 예문)

7.1 화면 네비게이션 맵

┌─────────────────────────────────────────┐ │ 앱 시작 │ └────────────────┬────────────────────────┘ ↓ ┌─────────────────┐ │ 로그인 확인 │ └────────┬────────┘ ↓ 토큰 있음? ├─ NO │ └─→ ┌──────────────────┐ │ │ LoginScreen │ │ │ [Spotify 로그인] │ │ └──────────┬───────┘ │ ↓ └─────────────┬───┘ ↓ ┌──────────────────────┐ │ HomeScreen │ │ (곡 목록) │ ┌──┤ [곡 선택] │ │ └──────────────────────┘ │ ↓ ┌──────────────────────┐ │ PlayerScreen │ │ (플레이어) │ │ - 가사 표시 │ │ - 플레이 컨트롤 │ │ - 단어 검색 │ ├──┤[← 뒤로] │ │ └──────────────────────┘ │ └──→ HomeScreen으로 돌아가기

7.2 화면별 구성 요소

LoginScreen

  • 앱 로고 및 설명
  • 기능 소개 (3개 항목)
  • Spotify 로그인 버튼
  • 주의사항 (30초 미리듣기 제한)

HomeScreen

  • 헤더 (제목 + 곡 개수)
  • BTS 곡 목록 (FlatList)
  • 각 곡 아이템:
    • 앨범 아트
    • 곡 제목
    • 앨범명
    • 재생 시간

PlayerScreen

  • 배경 (앨범 아트 + 블러)
  • 앨범 아트 (크게)
  • 곡 정보 (제목, 아티스트)
  • 가사 뷰 (스크롤 가능)
    • 현재 라인 하이라이트
    • 단어별 클릭 가능
  • PlayerControls
    • 재생 바
    • 재생시간 / 전체시간
    • 이전, 재생/일시정지, 다음
  • VocabCard (단어 선택 시)

10. 설치 및 실행 가이드

10.1 사전 요구사항

  • Node.js 18+ 설치
  • npm 또는 yarn 설치
  • Expo CLI: npm install -g @expo/cli
  • Spotify 계정 (무료)
  • Spotify Developer 앱 생성

10.2 단계별 설치

1단계: 프로젝트 클론

git clone https://github.com/ymi-hub/K-pop.git cd K-pop

2단계: 의존성 설치

npm install

3단계: 환경 설정

cp .env.example .env # .env 파일에 Spotify Client ID 입력

4단계: Spotify 설정

  1. https://developer.spotify.com/dashboard 접속
  2. Your App → Edit Settings
  3. Redirect URIs에 추가: http://localhost:19006/

10.3 앱 실행

웹 버전 (권장)

npm run web # http://localhost:19006에서 앱 열기

iOS

npm run ios # iOS 에뮬레이터에서 실행

Android

npm run android # Android 에뮬레이터에서 실행

11. 에러 처리 및 제한사항

11.1 주요 제한사항

항목 제한사항 이유
음악 재생 30초 미리듣기만 Spotify API 정책
로그인 웹 환경에서만 Spotify OAuth 제한
전곡 듣기 Spotify Premium 필요 Spotify 정책
가사 lrclib.net에 있는 곡만 API 의존성
네트워크 인터넷 필수 API 기반 앱

11.2 에러 처리 전략

API 호출 에러

try { const response = await fetch(API_URL); if (!response.ok) { throw new Error(`API Error: ${response.status}`); } return await response.json(); } catch (error) { console.error('Error:', error); // 사용자 친화적 메시지 표시 return []; }

토큰 만료 처리

  • 만료된 토큰 자동 삭제
  • 재로그인 요청
  • 로그인 화면으로 리다이렉트

11.3 일반적인 문제 해결

문제 해결 방법
"Client ID Invalid" .env 파일의 Client ID 확인 후 재입력
"redirect_uri not matching" Spotify Dashboard의 Redirect URIs 재확인
npm install 실패 node_modules 삭제 후 npm cache clean --force 실행
가사 안 나옴 lrclib.net에 해당 곡이 없을 수 있음
